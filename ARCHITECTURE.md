# PORM 企业级架构设计文档

## 1. 系统概述

### 1.1 项目定位
PORM（Poetic Rhythm Master）是一款基于 NLP + LLM 的企业级中文对联评分系统，采用字节跳动级工业标准设计，服务于对联创作、教学、比赛等专业场景。

### 1.2 核心特性
- **四层评分架构**：形式合规(30%) + 对仗技术(30%) + 艺术表现(30%) + AI印象(10%)
- **马鞍工程控制**：多层控制机制确保LLM输出可控、精确、可追溯
- **NLP-LLM融合**：基于数学模型的深度融合算法，相互制约协同
- **提示词外部化**：完整的配置管理系统，支持动态更新

### 2.2 模块职责

#### 2.2.1 接口层 (Interface Layer)
- **职责**：对外提供服务接口
- **模块**：REST API、CLI Tool、Python SDK、WebSocket
- **设计原则**：单一职责、接口隔离

#### 2.2.2 应用层 (Application Layer)
- **职责**：编排领域服务，实现业务用例
- **核心类**：`CoupletAnalyzer`
- **功能**：协调各维度分析，汇总评分结果

#### 2.2.3 领域层 (Domain Layer)
- **职责**：核心业务逻辑，包含马鞍工程、融合引擎、LLM分析器
- **核心组件**：
  - `SaddleEngineering`：多层控制机制
  - `FusionEngine`：NLP-LLM融合
  - `LLMAnalyzer`：LLM深度分析

#### 2.2.4 引擎层 (Engine Layer)
- **职责**：提供基础算法能力
- **组件**：
  - `MeterEngine`：诗律词谱匹配
  - `PingZeEngine`：平仄检测
  - `EmbeddingEngine`：向量嵌入

#### 2.2.5 基础设施层 (Infrastructure Layer)
- **职责**：提供技术支持服务
- **组件**：
  - `PromptConfigService`：提示词配置管理
  - `LoggingService`：日志记录
  - `CacheService`：缓存服务
  - `DataLoader`：数据加载

## 3. 马鞍工程详解

### 3.1 设计理念
"马鞍工程"比喻对LLM这匹"野马"的驾驭能力，通过多层控制机制确保输出可控。

### 3.2 控制架构

```
输入层控制 ──────────────────────────────────────────────────────────────
│
├── 预处理：标准化、清洗、格式化
├── 长度验证：2-100字符限制
├── 字符集验证：纯中文字符
└── 字数相等验证
│
过程层控制 ──────────────────────────────────────────────────────────────
│
├── 分数范围强制：0-100截断
├── 权重归一化：子分数加权平均
└── 一致性验证：分数与评价匹配
│
输出层控制 ──────────────────────────────────────────────────────────────
│
├── 分数边界验证
├── NLP-LLM一致性验证（差异<30%）
├── 格律合规验证
├── NLP修正：差异过大时加权融合
└── 格律修正：违规扣分
│
决策层 ─────────────────────────────────────────────────────────────────
│
├── 严格模式：违规数>0则失败
└── 决策链记录：完整可追溯
```

### 3.3 控制目标与评估指标

| 控制层级 | 控制目标 | 技术实现 | 评估指标 |
|---------|---------|---------|---------|
| 输入层 | 确保输入合法 | 正则验证、长度检查 | 违规率<0.1% |
| 过程层 | 确保过程可控 | 范围限制、一致性检查 | 异常率<1% |
| 输出层 | 确保输出准确 | 多源验证、自动修正 | 准确率>95% |
| 决策层 | 确保决策合理 | 严格模式、完整记录 | 可追溯率100% |

## 4. NLP-LLM融合引擎

### 4.1 融合策略

#### 4.1.1 加权平均融合（默认）
```
Final_Score = 0.4 × NLP_Score + 0.6 × LLM_Score
```

**NLP特征权重分配**：
- 语义相似度：25%
- 词嵌入相似度：20%
- 上下文相似度：10%
- 词性匹配率：15%
- 结构平行度：10%
- 句法一致性：5%
- 平仄模式得分：5%
- 韵律和谐度：5%
- 字符重叠率：3%
- 长度平衡度：2%

#### 4.1.2 贝叶斯融合
```
P(Score|NLP,LLM) ∝ P(NLP|Score) × P(LLM|Score) × P(Score)
```

### 4.2 特征工程

#### 4.2.1 语义特征
- BERT嵌入向量余弦相似度
- 逐字语义对齐分析
- 上下文语义一致性

#### 4.2.2 语法特征
- 词性标注匹配率
- 句法结构平行度
- 依存关系一致性

#### 4.2.3 韵律特征
- 平仄相对率
- 韵律和谐度
- 节奏匹配度

## 5. 提示词配置管理

### 5.1 系统架构

```
PromptConfigService (单例)
│
├── PromptConfig
│   ├── name: 提示词名称
│   ├── prompt_type: 类型枚举
│   ├── versions: 版本字典
│   └── active_version: 激活版本
│
└── PromptTemplate (不可变)
    ├── template: Jinja2模板
    ├── version: 语义化版本
    ├── parameters: 参数定义
    └── metadata: 元数据
```

### 5.2 配置示例

```yaml
name: duizhang_analysis
prompt_type: DUIZHANG_ANALYSIS
active_version: v1.0.0
versions:
  v1.0.0:
    template: |
      请从文学角度分析以下对联的对仗质量：
      上联：{{ upper }}
      下联：{{ lower }}
      ...
    parameters:
      required: [upper, lower]
      optional: [context, style]
    metadata:
      author: porm-system
      created_at: "2024-01-01"
```

### 5.3 动态更新机制
- 文件监听：30秒轮询检查
- 热重载：配置变更自动生效
- 版本回滚：支持快速回退
- A/B测试：灰度发布能力

## 6. 接口规范

### 6.1 Python SDK

```python
from porm import CoupletAnalyzer

# 初始化
analyzer = CoupletAnalyzer(
    api_key="your-api-key",
    base_url="https://api.example.com/v1",
    model="gpt-4"
)

# 分析
result = analyzer.analyze("春风化雨", "秋月寒霜")

# 结果
print(result.total_score)      # 总分
print(result.grade)            # 评级
print(result.formal_score)     # 形式合规得分
print(result.duizhang_score)   # 对仗技术得分
print(result.artistic_score)   # 艺术表现得分
print(result.impression_score) # AI印象得分
```

### 6.2 CLI 接口

```bash
# 对联评分
python -m porm.main couplet "春风化雨" "秋月寒霜" \
    --api-key $API_KEY \
    --base-url $BASE_URL \
    --model gpt-4

# 诗律检测
python -m porm.main shi "白日依山尽" "黄河入海流" \
    --pattern 五律

# 列出可用模式
python -m porm.main list shi
```

## 7. 容错机制

### 7.1 异常分类

| 异常类型 | 处理策略 | 降级方案 |
|---------|---------|---------|
| 输入非法 | 立即拒绝，返回错误 | 无 |
| LLM超时 | 重试3次，仍失败则报错 | 无 |
| NLP失败 | 记录日志，使用LLM结果 | 无 |
| 融合异常 | 使用单一来源结果 | 无 |

### 7.2 监控告警

- **指标监控**：QPS、延迟、错误率
- **日志记录**：结构化日志，全链路追踪
- **告警规则**：错误率>5%触发告警

## 11. 总结

PORM系统采用企业级架构设计，通过马鞍工程实现LLM输出的精确控制，通过NLP-LLM融合实现多源信息的科学整合，通过提示词外部化实现配置的灵活管理。整个系统无退化方案、无简化逻辑，所有算法均基于严格的数学模型，确保评分的科学性、准确性和可追溯性。

## 2. 项目背景

### 2.1 业务背景

OpenPROM 是一个面向中文古典诗词创作的纯 LLM 应用层助手。项目目标是让 AI 不仅能够"生成"诗词，还能按照传统诗联的格律、对仗、意境规则进行**可解释、可自修正**的评鉴与创作。

### 2.2 要解决的核心问题

1. **AI 生成诗词但不懂格律**：通用 LLM 对平仄、押韵、对仗规则掌握不稳定，容易产生"看起来像古诗但不合规"的内容。
2. **评分黑盒化**：现有诗词评分工具多依赖规则或纯 LLM，缺乏多层校准与可解释性。
3. **知识检索与创作脱节**：创作者需要快速检索相关古诗词作为灵感，但检索结果往往与生成/评鉴任务割裂。
4. **Agent 执行不可观测**：多轮工具调用、自修正过程缺乏持久化追踪。

### 2.3 项目目标

- 提供对联评分、生成、补全，律诗生成、补全，格律检测等核心能力。
- 通过"规则引擎作为 LLM Tool"的架构，让 LLM 自修正格律问题。
- 引入马鞍工程（Saddle Engineering）对 LLM 输出做多层质量控制。
- 提供可观测的 Agent 任务轨迹与知识检索流水线。

## 3. 产品定位

### 3.1 一句话描述

> 为中文诗词爱好者与创作者提供**可解释、可自修正、可检索**的 AI 诗词评鉴与创作助手。

### 3.2 目标用户

| 用户角色     | 描述                                 | 核心诉求                       |
| ------------ | ------------------------------------ | ------------------------------ |
| 诗词爱好者   | 希望快速评鉴对联/律诗质量            | 获得结构化、可解释的评分反馈   |
| 内容创作者   | 需要 AI 辅助生成符合格律的诗联       | 生成可格律自修正的对联与律诗   |
| 教育工作者   | 用于教学演示平仄、对仗、押韵         | 可视化逐字格律分析             |
| AI 研发人员  | 研究 LLM + 规则引擎的协作模式        | 可观测的 Agent 轨迹与工具调用  |

### 3.3 核心功能模块

| 模块名称     | 功能描述                                     | 优先级 | 状态     |
| ------------ | -------------------------------------------- | ------ | -------- |
| 对联评鉴     | 形式 / 技法 / 艺术 / 印象 四层评分 + 马鞍校准 | P0     | 已完成   |
| 对联生成     | 主题驱动 + 三阶段流水线 + 格律自修正         | P0     | 已完成   |
| 对联补全     | 给定上联，AI 补下联                          | P0     | 已完成   |
| 律诗生成     | 五绝 / 七绝 / 五律 / 七律 + 平仄偏好         | P0     | 已完成   |
| 律诗补全     | 残篇续写                                     | P0     | 已完成   |
| 格律检测     | 诗 / 词 / 联 的平仄、押韵、模板匹配          | P0     | 已完成   |
| 知识检索     | 五阶段 RAG 流水线（召回 / 融合 / 重排）      | P1     | 已完成   |
| 任务轨迹     | Agent 执行记录持久化与查询                   | P1     | 已完成   |

## 4. 项目范围

### 4.1 包含（In Scope）

- REST API 服务（FastAPI）与纯静态 SPA 前端。
- 对联 / 律诗的评鉴、生成、补全。
- 诗 / 词 / 联 的格律检测。
- 基于 ChromaDB + BM25 的混合知识检索。
- Agent 任务追踪持久化。
- Docker Compose 一键部署。

### 4.2 不包含（Out of Scope）

- 本地大模型部署（项目坚持远程 OpenAI 兼容 API）。
- 用户认证与权限管理（当前仅基于 session_id 匿名会话）。
- 社交媒体发布、支付、商业化运营功能。
- 多语言支持（当前仅中文）。

### 4.3 未来考虑（Future Consideration）

- 词牌生成与补全。
- 用户收藏与公开作品墙。
- 检索质量评估闭环与反馈入向量库。
- 更细粒度的 Saddle 工程可视化。

## 5. 关键约束

| 约束类型 | 描述                                         |
| -------- | -------------------------------------------- |
| 技术约束 | 必须兼容 OpenAI 兼容协议；不绑定特定模型厂商 |
| 性能约束 | 单次评鉴依赖 LLM 调用，响应时间受模型限制   |
| 数据约束 | 古诗词语料来自开源 chinese-poetry 库，需合规使用 |
| 部署约束 | 默认 SQLite，生产建议切换 PostgreSQL        |
